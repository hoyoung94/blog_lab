# 1주차 체크리스트

`blog_lab` 1주차 목표는 "프로젝트를 이해하고, 내 블로그로 바꾸고, 로컬에서 안정적으로 실행하는 것"입니다.

## 1일차 - 실행 확인과 구조 읽기

- `cmd /c npm install`
- `cmd /c npm run dev`
- 홈, 글, 소개 페이지를 직접 눌러보기
- `src/app`, `src/components`, `src/data`, `src/lib` 폴더 역할 메모하기

수정 대상:
- 없음. 오늘은 읽고 이해하는 데 집중합니다.

완료 기준:
- 각 폴더가 무슨 역할인지 1문장씩 설명할 수 있다.
- 로컬에서 블로그가 열린다.

## 2일차 - 블로그 정체성 맞추기

- 홈과 소개 페이지 문구를 내 학습 목표에 맞게 바꾸기
- 샘플 포스트를 내 주제로 교체하기

수정 대상:
- `src/app/page.tsx`
- `src/app/about/page.tsx`
- `src/data/posts.ts`
- `src/components/site-footer.tsx`

완료 기준:
- 누가 봐도 "클라우드와 리눅스를 배우는 학생의 개발 블로그"처럼 보인다.

## 3일차 - 메타데이터와 배포 준비

- 사이트 설명문 확인하기
- 나중에 배포할 도메인 자리에 임시 URL 넣어두기
- `sitemap`, `robots` 파일이 어떤 역할인지 이해하기

수정 대상:
- `src/app/layout.tsx`
- `src/app/sitemap.ts`
- `src/app/robots.ts`

완료 기준:
- 메타데이터가 무엇인지 설명할 수 있다.
- 어떤 값이 나중에 실제 배포 URL로 바뀌어야 하는지 알고 있다.

## 4일차 - Git 시작

- Git 저장소 상태 확인
- 첫 커밋 만들기
- GitHub 저장소 생성 후 푸시 준비

명령 예시:
- `git status`
- `git add .`
- `git commit -m "프로젝트 문구 한글화 및 학습용 블로그 설정"`

완료 기준:
- 커밋 1개 이상이 남아 있다.
- GitHub에 올릴 준비가 끝났다.

## 5일차 - 빌드 확인

- `cmd /c npm run build`
- 빌드 오류가 나면 메시지 읽고 고치기
- `cmd /c npm run lint`

수정 대상:
- 오류가 나는 파일

완료 기준:
- 로컬 빌드가 성공한다.
- 린트가 통과하거나, 어떤 경고가 남는지 이해하고 있다.

## 6일차 - 콘텐츠 1개 더 작성

- 새 글 1개 추가하기
- 글 제목, 요약, 태그를 내가 직접 정하기
- 글 내용은 짧아도 되지만 배운 내용을 기록하기

수정 대상:
- `src/data/posts.ts`

추천 주제:
- "Windows에서 npm 실행 정책 오류를 만난 뒤 해결한 과정"
- "왜 Vercel부터 시작하고 AWS는 나중에 하려는가"
- "Ubuntu 서버에 배포하기 전에 먼저 이해해야 할 것"

완료 기준:
- 내 경험이 담긴 글이 최소 1개 있다.

## 7일차 - 회고와 다음 주 준비

- 이번 주에 어려웠던 점 3개 적기
- 다음 주 목표를 `GitHub 업로드`와 `Vercel 배포`로 정리하기
- README에 실행 방법과 주의사항 갱신하기

수정 대상:
- `README.md`

완료 기준:
- 다음 주에 무엇을 할지 명확하다.
- 남이 봐도 이 프로젝트를 실행할 수 있는 설명이 있다.
